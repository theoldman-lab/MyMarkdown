# DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning

## 理论动机：为何要探索“纯 RL”驱动推理？

### 当前后训练范式的瓶颈

当前主流 LLM 后训练流程通常遵循：预训练 → SFT（监督微调） → 奖励建模（RM） → 强化学习（RLHF/RLAIF）

其中，SFT 是不可或缺的“冷启动”步骤，依赖大量高质量标注数据（如人工编写的 CoT）。然而：

- **数据获取成本高**：优质长 CoT 数据需专家撰写或复杂合成。
- **泛化能力受限**：SFT 数据分布决定了模型上限，难以超越训练集覆盖范围。
- **策略固化风险**：模型容易模仿固定解题模板，缺乏自主探索能力。

### 强化学习作为“智能涌现引擎”的潜力

DeepSeek 团队提出一个根本性假设：
> **LLM 具备内在的推理潜力，只需通过合适的奖励信号引导，即可在无监督条件下自我演化出高效的问题解决策略。**

这一假设源于认知科学中的“试错学习”理论——人类智能并非全靠教导，而是在反馈中不断调整行为。若能在 LLM 中复现此过程，则可突破 SFT 的天花板。

## DeepSeek-R1-Zero：纯强化学习的技术实现细节

### 基础模型选择：DeepSeek-V3-Base

- **未经过任何指令微调**，保留原始语言建模能力。
- 参数量级为百亿级别，具备足够容量支持复杂推理。
- 选择基础模型而非 Instruct 模型，是为了避免先验偏置干扰 RL 的自然探索。

### 奖励函数设计：规则驱动 vs 神经网络 RM

#### 为何放弃神经网络奖励模型？

尽管 RM 在对齐任务中广泛应用，但在纯推理场景存在三大问题：

| 问题                           | 说明                                          |
| ------------------------------ | --------------------------------------------- |
| **奖励黑客**（Reward Hacking） | 模型学会欺骗 RM（如生成看似合理但错误的 CoT） |
| **评估滞后**                   | RM 需持续更新以匹配策略模型，增加训练复杂度   |
| **泛化偏差**                   | RM 在训练分布外问题上不可靠（如新数学定理）   |

因此，DeepSeek 采用**完全确定性的规则奖励系统**：

#### 准确性奖励（Accuracy Reward）

- **数学题**：
  - 要求最终答案格式为 `\boxed{...}`。
  - 使用 SymPy 或数值验证器判断等价性（如 `2+2=4` 与 `4` 视为等价）。
  - 对开放题（如证明题），采用多步验证：中间结论是否逻辑自洽。
- **编程题**（LeetCode 风格）：
  - 编译代码并运行预设测试用例。
  - 奖励 = 通过测试数 / 总测试数。
  - 支持超时与内存限制，防止无限循环。

#### 格式奖励（Format Reward）

- 强制结构：`<think> ... </think>` 包裹推理过程。
- 若缺失标签或格式错误，奖励直接归零。
- 此设计确保后续可解析 CoT，便于拒绝采样与蒸馏。

#### 语言一致性奖励（仅用于 DeepSeek-R1）

- 计算 CoT 中非目标语言字符比例（如中文提示下出现英文段落）。
- 奖励 = 1 - α × (非目标语言比例)，α 为调节系数（实验设为 0.5）。
- 实验表明：该奖励使语言混合率下降 80%，可读性显著提升。

### 提示模板设计：最小干预原则

为避免引入人为偏置，提示模板极为简洁：

- **不指定解题方法**（如“使用归纳法”）。
- **不强制反思**（如“检查你的答案”）。
- 目标：让模型在 RL 过程中**自主发现**最佳策略。

## DeepSeek-R1：四阶段训练管线详解

### 阶段 1：冷启动 SFT（Cold Start SFT）

#### 数据构建策略

| 来源              | 数量  | 特点                         |
| ----------------- | ----- | ---------------------------- |
| Few-shot 提示生成 | ~2000 | 使用 o1 风格提示引导长 CoT   |
| R1-Zero 优质输出  | ~1500 | 人工筛选正确且结构清晰的响应 |
| 人类标注优化      | ~500  | 修正语言、添加总结、统一格式 |

#### 格式标准化

采用双分隔符结构：
```
<sot><reasoning>...</reasoning><eot><summary>...</summary>
```
- `<sot>/<eot>` 为特殊 token，便于模型学习边界。
- `<summary>` 必须简洁、独立可读，不含技术细节。

### 阶段 2：推理聚焦 RL

- **训练数据分布**：90% 推理任务（数学/代码/逻辑），10% 通用任务。
- **批大小**：8192（极大 batch 提升梯度稳定性）。
- **KL 散度约束**：防止策略偏离初始模型过远（KL coeff = 0.01）。
- **收敛判断**：AIME 验证集 pass@1 连续 500 步波动 < 0.5%。

### 阶段 3：拒绝采样 + 多领域 SFT

#### 推理数据生成流程

1. 从 RL 检查点采样 16 个响应/提示。
2. 应用规则过滤器：
   - 语言一致性 ≥ 90%
   - 无冗余重复段落
   - CoT 长度 ∈ [200, 2000] tokens
3. 对无法规则验证的问题（如开放证明），使用 **DeepSeek-V3 作为裁判模型**：
   - 输入：(问题, 标准答案, 模型响应)
   - 输出：是否正确（二分类）
   - 该过程称为“生成式奖励建模”，但仅用于数据筛选，不参与 RL。

#### 非推理数据处理

- 对写作、角色扮演等任务，**有条件生成 CoT**：
  - 复杂查询（如“写一篇关于量子计算的科普文章”）→ 生成大纲式 CoT。
  - 简单查询（如“今天天气如何？”）→ 直接回答，避免冗余。
- 此设计平衡了“推理普适性”与“响应效率”。

### 阶段 4：全场景 RL

#### 奖励信号融合

| 任务类型  | 奖励来源 | 评估对象           |
| --------- | -------- | ------------------ |
| 数学/代码 | 规则奖励 | 完整响应（含 CoT） |
| 写作/QA   | 神经 RM  | 仅 `<summary>`     |
| 安全敏感  | 安全 RM  | 完整响应           |

- **有用性 RM**：基于 DeepSeek-V3 的偏好数据训练，关注信息量、相关性。
- **无害性 RM**：检测偏见、歧视、非法建议等，覆盖 CoT 中的潜在风险。

#### 训练技巧

- **课程学习**（Curriculum Learning）：初期侧重推理任务，后期逐步增加通用任务比例。
- **动态权重调整**：根据验证集表现自动平衡各奖励项权重。
